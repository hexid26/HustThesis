\zhabstract{
% 论文针对移动群智感知中感知、通信、计算三个重要环节


移动群智感知利用众包的方式将数据采集任务分发给无处不在的智能移动设备，通过设备内集成的各种传感器收集感知数据。
% 相较于传统的无线传感网络，移动群智感知的执行效率更高、适用范围更广、部署成本更低。
% 早期阶段，移动群智感知常借助云计算技术处理大量的感知数据。
% 如今，万物互联的时代已经来临，成倍增加的移动设备，不仅扩大了移动群智感知的覆盖范围，也让感知数据的体积上升到更大的数量级。
边缘计算作为与云计算模型互补的分布式计算模型，具备覆盖范围广、涵盖设备多以及耦合多种异构网络，这些特性为移动群智感知质量优化和效率优化提供了新的契机。
另一方面，边缘网络中的移动设备利用端到端（Device-to-Device，简称D2D）通信，不仅有助于降低感知过程中的数据转发成本，还可以减少移动群智感知对主干网络资源的占用。
如何有机结合边缘计算和 D2D 通信实现大范围、低延时、少成本的移动群智感知应用，已成为目前亟待解决的重要课题。

\textbf{现有研究的问题整理}

为了扩大感知数据采集范围、提高收集效率、降低执行成本，可以合理使用边缘资源和 D2D 通信对移动群智感知应用的三个重要过程进行优化：即感知、数据处理和通信。
在感知过程中，由于感知数据存在一定的时效性和地域分布特性，因此需要在指定时间范围内分发感知任务并收集相关数据，以保障感知数据的有效性。
因此，如何借助边缘资源和 D2D 通信提高目标范围内的感知任务部署效率和感知数据收集效率是感知过程中的关键工作之一。
在通信过程中，移动群智感知应用多以 D2D 通信为主、蜂窝通信为辅的网络结构进行数据交换，以减轻感知数据对主干网络资源的占用。
另一方面，由于感知设备的计算能力参差不齐，性能不足的感知设备需要通过计算任务卸载技术借助外部算力对感知数据进行处理。
面对大量的网络通信需求，如何利用高效便捷的网络资源管理技术，例如软件定义网络技术（Software-Defined Networking，简称SDN）来调度网络资源、合理规划感知数据传输路径、减少感知数据传输时延和处理时延，也是优化群智感知执行效率的关键工作之一。
在数据处理过程中，移动群智感知多采用云端融合的思想，借助边缘服务处理大量的感知数据。

移动群智感知应用包含三个重要过程：感知、通信和数据处理。
在感知过程中，%由于移动设备并不具备感知任务的发现能力，
可以借助边缘服务、 D2D 通信等手段加速感知任务的分发和感知数据的收集。
因此，研究边缘服务、 D2D 通信对群智感知执行效率的影响机理，是合理制定群智感知应用优化策略、提高群智感知执行效率和感知质量的关键工作之一。
% 如何利用边缘服务器和 D2D 通信提高感知任务的执行效率、扩大感知服务的覆盖范围，是提高感知质量的关键工作之一。
在通信过程中，为了减少通信成本开销和主干网资源占用，移动群智感知中的数据交互多以 D2D 通信为主、蜂窝通信为辅。
因此，移动群智感知需要大量的网络链接来传输离散分布的感知数据。
如何借助高效、便捷的网络资源管理技术，例如软件定义网络技术（Software-Defined Networking，简称SDN）来调度网络资源、合理规划传输路径，也是提高感知任务执行效率，减少执行成本的关键研究工作。
在数据处理过程中，移动群智感知多采用云端融合的思想，借助边缘服务处理大量的感知数据。
面对智慧城市感知等大范围群智感知应用，如何针对感知任务的需求部署边缘服务器、对感知数据进行处理和收集，也是提高移动群智感知效率的一项重要研究工作。
围绕移动群智感知应用三个过程中所面临的主要问题，需要深度挖掘边缘计算中计算资源、网络资源以及 D2D 通信在移动群智感知应用中的影响作用和机理，对面向群智感知的边缘计算资源调度优化展开研究。


% 『面对海量的感知数据传输，如何借助软件定义网络技术（Software-Defined Networking，简称SDN）合理调度网络资源、规划传输路径，也是提高感知任务执行效率，减少执行成本的关键研究工作。』
% 将数据处理工作委托给边缘服务器，以达到加速最终数据处理的目的。

针对感知过程
%中移动设备的运动状态对D2D通信造成的影响
，提出了基于感知质量的任务调度机制，以提高感知任务在多任务传播源、多数据接收源场景下的执行效率。
% 由于D2D通信的引入，感知任务的部署和移动设备的运动状态密切相关。
目前的分析方法，无法准确描述移动群智感知应用的完整执行过程。
基于此现状，利用随机过程分析将感知执行过程转化为常微分方程组，用以描述感知任务的执行过程。
借助随机过程分析，找出了移动设备在机会式通信网络中影响感知任务传播速度和感知数据收集速度的主要因素，例如移动设备的相遇率、边缘服务的部署数量、边缘服务和移动设备的相遇率、群智感知生命周期等。
利用构建的理论模型，设计了二阶段资源决策算法，制定了感知任务的分发策略和感知任务执行时间划分方法，在感知任务生命周期内提高移动群智感知的感知效率。同时，对不同的影响因素提出优化策略与意见。

针对通信过程，提出了面向计算任务卸载的网络资源调度机制。
移动群智感知中，感知设备可以使用大量的 D2D 通信来减少通信成本，同时还需要可靠的链接来保障性能较弱的感知设备与边缘服务之间的感知数据计算任务卸载。
因此，可以使用 SDN 技术对端到端、端到边的大量网络链接进行管理和调度。
在网络资源调度过程中，不仅需要考虑带宽、延时的制约，面对海量的感知设备时，SDN 交换机的流表容量也会对链路的调度产生约束。
% 提出了基于计算任务卸载的网络资源调度机制，利用SDN优化边缘网络中移动设备的通信效率。
% 在移动群智感知应用中，%移动设备的数据处理能力、网络通信能力存在个体差异。
% 对于能力有限的移动设备，可以将原始感知数据直接交给边缘服务器处理。
% 然而，诸如视频、图像类的原始感知数据体积较大，容易导致边缘网络中的通信资源失衡，甚至引起网络堵塞。
基于此现状，利用排队论模型描述了端设备和边缘设备上的计算任务处理过程，建立了计算卸载决策、能耗以及链路资源之间的整数线性规划模型。
通过对该模型的学习和分析，在保障感知数据计算任务卸载收益最大化的同时，实现了边缘网络中的链路负载均衡。

针对数据处理过程，提出了基于城市居民移动特征的边缘服务调度机制，解决在城市范围内实施移动群智感知时边缘服务的部署问题。
为了获取城市主体和物理对象的数字信息，智慧城市研究主要借助移动群智感知来收集基础数据。
% 利用人类生活中无处不在的移动设备，移动群智感知可以提取城市主体和物理对象的大量数字信息，作为智慧城市研究的基础数据。
在该应用场景下，通过借助城市居民的移动特征来部署边缘服务，可以判断出移动设备聚集的时机和区域，并合理部署边缘服务，以更快捷地处理、收集感知数据，并提高 D2D 通信的利用率。
% 提高移动群智感知的服务质量。
基于此现状，该机制借助城市中已有的公交网络和公共交通中的乘客流动特征，建立了乘客轨迹、公交路线和边缘服务部署决策的混合整数线性规划模型。
通过对该模型的分析求解，降低了城市感知应用中的边缘服务部署成本。

综上所述，围绕移动群智感知的三个主要过程，利用边缘计算资源调度优化机制可以减少数据传输、数据处理的时间，在保障感知质量的前提下合理降低能耗成本和服务成本。
同时，与D2D通信的深度结合，为移动群智感知技术在 5G 时代中的运用，打下坚实的基础。

\enabstract{

Mobile crowdsensing distributes sensing tasks to ubiquitous mobile devices by crowd, through collecting sensing data from various integrated sensors inside of devices. 
However, it is challenging to achieve wide coverage mobile crowdsensing with low latency. 
Edge computing, as a complementary distributed computing model to cloud computing model, provides new opportunities for supporting mobile crowdsensing, due to its wide coverage, prevalent devices, and heterogeneous network infrastructure.
On the other hand, D2D (\emph{Device-to-Device}) communication becomes a promising way to reduce the cost of data interaction during the sensing process among mobile devices.
Therefore, there have been increasing research interests on integrating edge computing and D2D communication techniques for mobile crowdsensing.

Mobile crowdsensing application contains three crucial processes: sensing, communication, and data processing.
In the sensing process, mobile crowdsensing task initiator generally employ edge servers, D2D communication and so on to accomplish sensing task dissemination and sensing data collection.
One of the most important works of enhancing the quality of sensing is figuring out how to use edge servers and D2D communication to improve the efficiency of sensing task and enlarge the sensing coverage.
In the communication process, data interaction in mobile crowdsensing mainly adopts D2D communication supplemented by cellular communication, in order to decrease the communication overhead and the occupation of backbone network resource.
Faced with massive amounts of sensing data transmission, it's also an important research to work out how to use SDN~(\emph{Software-Defined Networking}) reasonable schedule network resource and plan transmission path to improve the efficiency of sensing task and reduce the cost.
In the data processing process, edge computing model is widely used in mobile crowdsensing, processing massive sensing data by edge servers.
We propose several resource schedule and optimization technologies to enhance the quality of sensing, and to solve problems above in a conquer-and-divide manner. 

First, in the sensing process, we propose tasks schedule technology based on quality of sensing, to enhance the efficiency of sensing tasks in multiple source and receivers.
However, the existing analytical methods couldn’t describe the complete execution process of mobile crowdsensing precisely.
Thus we convert the sensing process into mathematics using stochastic analysis, and define the sensing process using ODEs~(\emph{Ordinary Differential Equations}).
This technology finds out the major factors of influence sensing tasks propagate speed and sensing data collect speed in mobile devices in D2D communication networks through math analysis. 

Second, in the communication process, we propose a network resource scheduling mechanism based on task offloading, and use SDN to optimize the communication efficiency of mobile devices in edge networks.
For mobile devices with limited capabilities, the original sensing data can be directly handed over to the edge server for processing.
However, the volume of raw sensing data, such as video and image, is huge, which easily leads the network congestion.
Based on this situation, we build an ILP~(\emph{integer linear programming}) model between offload decision, energy consumption, and network resource which is established.
Through the study and analysis of the model, we archieve the load balance of links in the edge network and maximize the benefits of computing task offloading.

Third, in the data processing, we proposes an edge service scheduling mechanism based on the urban social network to solve the deployment problem of edge service  for mobile crowdsensing in cities. 
Under this scenario, edge services are usually deployed with the help of the social network characteristics of urban residents to improve the sensing quality of mobile crowdsensing.
Based on this situation, the mechanism builds a ILP model with passenger trajectory, bus lines, and edge service deployment decisions by using the existing bus network and passenger flow characteristics in public transportation.
Through the analysis of this model, the deployment cost of edge services in urban mobile crowdsensing applications is reduced.

In summary, focusing on the three main processes of mobile crowdsensing, we use edge resource scheduling optimization technologies to reduce the time of data transmission and data processing, and reasonably reduce the energy consumption and service cost, and ensuring the sensing quality. 
At the same time, the deep combination with D2D communication also makes mobile crowdsensing more suitable for IoT scenarios.
}
